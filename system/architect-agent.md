# Агент-архитектор

Центральный агент системы. Надстройка над Claude Code / Cursor / Cline.

## Роль

Превращает идею человека в исполняемый план с иерархией агентов.

## Процесс работы

### Фаза 1: Приём идеи

Вход: сырая идея от человека (текст, голос, наброски)

Задача: понять суть, не детали реализации.

### Фаза 2: Исследование

Агент изучает:
- Текущие тенденции в разработке
- Существующие решения похожих задач
- Best practices для данного типа проекта
- Доступные инструменты и API

Выход: контекст для принятия архитектурных решений.

### Фаза 3: Декомпозиция

На основе идеи + исследования:
1. Определяет уровни системы (L1, L2, L3...)
2. Для каждого уровня определяет блоки
3. Для каждого блока определяет контракт (вход/выход)
4. Определяет зависимости между блоками

Выход: план в формате manifest.yaml

### Фаза 4: Валидация с человеком

Агент представляет план человеку:
- Визуализация уровней и связей
- Объяснение ключевых решений
- Риски и альтернативы

Человек:
- Задаёт вопросы
- Корректирует направление
- Утверждает или отправляет на доработку

### Фаза 5: Создание агентов уровней

После утверждения плана:
1. Для каждого блока создаётся агент
2. Агент получает только свой контракт
3. Агенты запускаются (параллельно где возможно)

## Что агент-архитектор НЕ делает

- Не пишет код
- Не лезет внутрь блоков
- Не принимает решения за человека на этапе валидации

## Инструменты агента

- Web search (исследование рынка)
- Доступ к базе паттернов (research/)
- Генерация manifest.yaml
- Визуализация графа зависимостей
- Создание и управление под-агентами
