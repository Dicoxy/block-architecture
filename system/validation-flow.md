# Поток валидации

Как человек взаимодействует с системой.

## Принцип

Человек валидирует ПЛАН, не код.

Человек не читает код, не ревьюит реализацию. Только:
- Понимает ли система идею правильно?
- Согласен ли с декомпозицией?
- Есть ли критичные упущения?

## Точки валидации

### 1. После исследования

Агент-архитектор показывает:
- Что нашёл по теме
- Какие аналоги существуют
- Какие подходы рекомендует

Человек:
- Корректирует направление если нужно
- Добавляет контекст который агент не знает
- Подтверждает понимание задачи

### 2. После декомпозиции

Агент-архитектор показывает:
```
Уровень 1: [Блок A] [Блок B]
                ↓
Уровень 2:   [Блок C]
              ↙   ↘
Уровень 3: [Блок D] [Блок E]
```

Плюс для каждого блока:
- Что на входе
- Что на выходе
- Зачем этот блок нужен

Человек:
- Проверяет логику разбиения
- Может объединить/разделить блоки
- Может изменить зависимости
- Утверждает или отправляет на переработку

### 3. При критических ошибках (runtime)

Если агент уровня не может справиться:
- Система показывает: какой блок, какая ошибка, что пробовали
- Человек решает: изменить контракт, дать подсказку, или отменить блок

## Формат представления плана человеку

Не код. Не yaml. Визуализация + простой язык.

```
┌─────────────────────────────────────────┐
│  ПРОЕКТ: Телеграм-бот для голосовых     │
├─────────────────────────────────────────┤
│                                         │
│  [Приём голоса] → [Транскрипция]        │
│                         ↓               │
│                   [Обработка LLM]       │
│                         ↓               │
│                [Постинг в соцсети]      │
│                                         │
├─────────────────────────────────────────┤
│  Блоков: 4                              │
│  Параллельных веток: 1                  │
│  Оценка сложности: низкая               │
└─────────────────────────────────────────┘

Утвердить план? [Да] [Изменить] [Отмена]
```

## Чего НЕ показываем человеку

- Код внутри блоков
- Технические детали реализации
- Промежуточные состояния агентов
- Логи работы (если не запросил)

Человек видит только:
- План (до запуска)
- Статусы блоков (во время работы)
- Финальный результат (после завершения)
